Principles

Keep It Simple

We try to keep code base at the bare minimum while still offer high productivity

No Code bloat

No monkey patch, or anything that touch the Core Classes, no main space method bloats, or universal global helpers.

Bare minimum run time processing

The framework it self try to preprocess anything that could be, to achieve minimal processing during request

Fully BDD

For creating production ready code, we require tdd, and for that, the framework made in mind with Behaviour Driven Design

You can create integration test with only be require the Test module in your specification.

Easy to Learn

Before we started to build this framework, we had the chance to use a lot of more or less popular framework, and the Sinatra DSL was one of the inspiration to go with in terms of DSL use.

While this is a choose in term of taste, the whole framework could be easily understand, all you need is your classic ruby method listing trick:

puts(self.methods - Object.methods)

Everything should be self describing. If not please do not hesitate tell us about your opinion

Principle Of Least Surprise

Simply as it is. Everything works just as how it is specified, no black magic included to correct even the developers mistake.

If something should came out during development, it will.

Modular design

As long you don't need something other than the core features, you are not forced to use a full development stack.

But you can easily extend the feature set then you require.

Writing plugins and isolated mountable applications is also cup cake

Only dependency to rule them all

Other than a webserver, all you need is rack, nothing more, to start create web applications

Open development

You can contribute to the project on Github anytime you want, even ideas are welcome.

Learning through

We Try to create Example Codes for every feature so it can be easily learned
